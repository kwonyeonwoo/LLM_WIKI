# SDLC And Spec-Driven Development

## Definition

Spec-driven development is the discipline of defining why, what, and how before implementation. In the Agentic Coding Basics lectures, it appears as the bridge between AI-assisted generation and reliable system development.

## SDLC Mapping

| SDLC Step | Agentic Artifact |
| --- | --- |
| Planning | system request, feasibility, project sponsor, business value |
| Analysis | PRD, SRS, stakeholder expectations, constraints |
| Design | architecture, interfaces, data flow, ADR |
| Implementation | code, tests, deployment |
| Validation | acceptance criteria, test reports, review documents |

## Why It Matters For LLM Wiki

An LLM Wiki is itself a spec-driven artifact. Raw materials enter the source layer, then agent instructions and schema decide how they become wiki pages. Without schema, the wiki turns into loose notes; without source notes, it loses evidence.
