# Inbox

아직 처리하지 않은 Markdown 자료를 넣는 곳이다.

## 사용법

1. 웹문서, 논문, 메모, 회의록 등을 Markdown으로 변환해 이 폴더에 넣는다.
2. 파일명은 가능하면 `YYYY-MM-DD-short-title.md` 형식으로 둔다.
3. LLM에게 "`inbox/파일명.md`를 ingest 해줘"라고 요청한다.
4. 처리가 끝난 자료는 필요하면 `sources/`에 출처 노트로 정리하고, 원본은 보관하거나 삭제 여부를 사용자가 결정한다.

## 주의

민감한 개인정보, API 키, 비공개 회사 자료를 넣기 전에 공유 범위와 백업 위치를 확인한다.
