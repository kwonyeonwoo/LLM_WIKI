## Summary

This lecture defines harness engineering as designing the execution environment around an AI agent: tools, data, verification pipelines, safety mechanisms, and work records. The key distinction is between prompt quality and operational structure.

The lecture introduces four responsibilities: contract, procedure, journal, and preference. Contract defines what counts as done, procedure defines how work proceeds, journal records what happened, and preference defines working style. It also connects this to `TASK.md`, skills, logs or MCP, and files such as `AGENTS.md`.

## Key Claims

- A procedure without a contract can create an infinite loop.
- A contract without a procedure is inert because the agent lacks an execution path.
- Done conditions should be measurable and small.
- Agents should work one checklist item at a time, verify, record, and stop.
- Common agentic patterns include prompt chaining, routing, parallelization, orchestrator-workers, and evaluator-optimizer.
