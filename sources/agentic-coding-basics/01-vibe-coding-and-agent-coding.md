## Summary

This lecture frames software engineering as system work rather than isolated code writing. A system is presented as interacting components, goals, boundaries, constraints, and rules. System analysis converts user, business, and operational goals into verifiable models and specifications.

The lecture contrasts traditional programming with AI-assisted development. Vibe coding is described as natural-language goal and constraint expression where the developer focuses on blueprint design, prompting, verification, and debugging rather than manual syntax. Agentic coding is introduced as the next step: coding agents can participate in SDLC-like processes, but only if work is structured, verified, and connected to requirements.

## Key Claims

- Coding is only one small part of the software development lifecycle.
- AI can generate code quickly, but system quality still depends on requirements, boundaries, validation, and responsibility.
- Vibe coding lowers implementation friction but can accumulate technical debt when used as "accept all" generation.
- Agentic coding becomes useful when it is tied to SDLC artifacts such as PRD, SRS, use cases, behavior models, ADRs, prototypes, and validation.
